审查指定的 Pull Request。

**用法：**
- `/review <pr-number>` - 审查指定的PR
- 例如：`/review 369`

**执行步骤：**

1. **获取PR信息**
   ```
   gh pr view <pr-number>
   gh pr diff <pr-number>
   gh pr checks <pr-number>
   ```

2. **分析PR内容**
   - 检查PR标题是否符合规范
   - 检查PR描述是否完整填写了模板
   - 审查代码变更的合理性
   - 检查是否有潜在的安全问题
   - 验证测试覆盖情况

3. **检查模板合规性**
   - 对照 `.github/pull_request_template.md` 检查必填项
   - 确认所有检查项都已勾选
   - 验证变更类型选择是否正确

4. **生成审查报告**
   - 列出发现的问题
   - 给出改进建议
   - 标注需要特别关注的地方

5. **可选：添加审查评论**
   - 如果用户同意，可以使用 `gh pr comment` 添加评论
   - 提供建设性的反馈

**审查要点：**
- ✅ PR 标题格式正确
- ✅ 描述完整填写了模板
- ✅ 变更类型选择正确
- ✅ 测试覆盖充分
- ✅ 代码质量良好
- ✅ 没有安全隐患
- ✅ 文档已更新
- ✅ 向后兼容性考虑
